FOSDEM Pentabarf HOWTO

This document will give you a short rundown of how to add speakers and sessions in Pentabarf for FOSDEM.

Login

First of all, log in to penta.fosdem.org. You should have received credentials to do so from the devrooms team. If not, please contact them.

First screen

You will be presented with a first screen that only contains information that is of importance to the core staff team.

The more important bits, for you, are to be found in the middle of the navigation bar on the left:

  • the first row of icons (with a looking glass) are for search
  • the second row of icons (with a + sign) are for creating
  • the first icon from the left is for Person instances, that is, speakers
  • the icon in the middle is for Event instances, that is, talks
  • the third icon is for Conference, something you do not have access to

Adding a speaker

You should add speakers first, either all of them, or on a talk-by-talk basis, that is up to you.

Later on, when you create Events, you can then assign Persons as speakers in those Events. The opposite does not work: while you can see which Events a Person is associated with in the Person screen, and navigate there, you cannot assign a Person as a speaker to an Event from the Person screen.

Note that the database contains Persons who have been speakers in FOSDEM 2011 as well and, hence, please first search whether a Person already exists.

Searching for an existing speaker

The most convenient option is to use the search text box that is above the icons that have been presented above.

Note that it searches for both Events and Persons.

If you do find a match, click on the Person and edit data there instead of creating a new Person, but the screens to fill out will be exactly the same as explained below.

Creating a new Person

Click on the icon as depicted on the left to create a new Person.

This is the screen you will be presented with when creating a new Person. Note that Pentabarf, for some odd reason, keeps in mind which tab (on top) you have last selected, so you might end up in a different tab than "General". If that is the case, please click on the "General" tab first.

This is what the General section should look like once it is filled out.
Please note that the relevant fields that must be filled out here are as follows:

  • First Name
  • Last Name
Everything else on this screen is optional.

If the speaker provided a photo, you can upload it by clicking on the Browse... button next to the Image field, and specify the file there.

Note that the website expects the photos to be at the very least square, and ideally 128 x 128 pixels. You may need to use something like Gimp, Krita, or the likes, in order to cut out the head and resize accordingly.

We typically use Gimp's Crop Tool for that purpose.

It is also preferred to use the PNG format for images, although JPEG and GIF are supported as well.

Important: please do also enable the Image public? checkbox in order for that picture to be exported and visible on the website.

Then, click on the Description tab on top to enter the biography of the speaker and fill out the following fields:

  • Abstract is mandatory and should contain a brief bio of the speaker
  • Description is optional and may contain additional information that did not fit into the Abstract
Note that we do not include the speaker bios in the booklet for devroom speakers and, hence, it does not matter much whether you put everything into the Abstract or whether you split it up. The website displays both, first the Abstract and then the Description.

Typically, we usually put everything into the Abstract field ;)

Once you have clicked the Save button as it appears in the bottom right corner of your browser window as soon as you have changed anything, the photo will be uploaded (if applicable), the data be persisted, and the page be reloaded.

Adding a talk

Once a speaker (or all speakers) of a talk, or all speakers of all talks (as you prefer) have been created, you can now add their talks to the conferencing system.

You should do so by clicking the new event button on the left, and as depicted here.

You will then be presented with a blank entry for an Event.
As for creating new Person instances, Pentabarf might open that page with a different tab than General. If that is the case, please click on the General tab above.

Data must be entered as follows:

  • Event title: quite obviously, the title of the talk: please keep it rather short, as a long title will result in quite a few issues for displaying it on the schedule and the booklet (remember that we have 300 to 400 sessions to cram in there ;))
  • Subtitle of the event: is not used anywhere on the website nor the booklet, hence to be kept empty
  • Slug: will be used for the URL of the talk on the FOSDEM website; e.g. a slug welcome_to_fosdem will result in the URL of that talk to be https://fosdem.org/2012/schedule/event/welcome_to_fosdem
    For obvious reasons (readable URLs), please restrain to lower case ASCII here and use _ (underline) to replace whitespaces (welcome_to_fosdem is definitely nicer than Welcome%20to%20FOSDEM in an URL)
  • Event Origin: does not matter for devrooms and is only for information purposes, so just leave it to its default value, idea, unless you want to keep track of that information yourself
  • Event state: must be set to accepted or it will not show up on the schedule
  • Progress: must be set to reconfirmed or it will not show up on the schedule
  • Team: not used, leave empty
  • Submission of presentation slides: not used either, leave to unknown unless you want to use it to keep track of that information yourself
  • Track: must be set to the name of your devroom, quite obviously
    Note that your devroom has been added to the list of tracks by the FOSDEM staff already. If that is not the case, please contact us.

The next step here is to define the speaker(s) for that Event.
To do that, please click on the Persons tab on top of the page.

Notice that your name has automatically been filled in as the submitter: you may very well leave that information there as it is, we don't use it anywhere on the website or on the booklet.

Click on the Add person button in order to add one or many people with the role of speaker to the Event.

After clicking the Add person button, an additional row has been added to the list of Persons.
If you wish to add several speakers to an Event, you may of course repeat that operation at will. Every Person with the Event role set to speaker will appear on the schedule (website and booklet).

As depicted on the left, make sure to select speaker as the role of the Person.

Once you have selected speaker as the Event role, an additional selection list will appear on its right, for Role state.
That field must be set to confirmed in order for the talk to show up in the schedule.

The next step is to click on the Description tab in order to fill out the abstract of the talk.

The following fields must be filled out:

  • Abstract: is mandatory; please keep it somewhat shorter as it is the part that will be included in the booklet
  • Description: is optional and may be used as an overflow if the abstract of the talk is somewhat longer
The rationale here is that the Abstract shows up both in the booklet and on the website, but the Description is only displayed on the website.
Note that on the website, both are displayed, the Abstract first, and the Description right below, so do not repeat the content of the Abstract in the Description field.

The last step is to specify the schedule information for that talk. In order to do so, please click the Schedule tab on top of the window.

Fill out all fields as follows:

  • Day: select Saturday or Sunday, accordingly to when your devroom has been planned (as you have been informed of by the devrooms team) or, if your devroom spans both days, as you have chosen to put it on your schedule
  • Start time: self-explanatory
  • Duration: self-explanatory
  • Room: please select the room that has been assigned to your devroom, as you have been informed of by the devrooms team
  • Public event: must be enabled to show up on the schedule

As the last step, press the Save button in the lower right corner of your browser window to persist the changes.

Exporting

Changing data in Pentabarf does not result in a change of the schedule on the fosdem.org website.
The reason is that we use an export process in between the two in order to integrate the scheduling information from Pentabarf properly into our website.

Triggering an export is something that only the devrooms team can do and, hence, once you have entered speakers and talks and would like that information to be put on our website, please contact us at devrooms@fosdem.org.

Note that if your schedule is still in flux, you may want to make sure that your Events are set to unconfirmed until it is final, or else it will be on our website once we trigger the export (e.g. upon request by another devroom).
